Yes, if you want API-level accuracy without paying for an API key, running an open-weights LLM locally on your own computer is exactly how you do it.

The Difference: API vs. Local LLMs
Paid APIs (OpenAI, Gemini API): You are paying to rent their massive supercomputers to process your text. You send the text over the internet, they process it, and send the summary back.

Free Local LLMs (Llama 3, Qwen): Tech giants like Meta (Facebook) and Alibaba spent millions training massive AI brains. Then, they released the "weights" (the files that make up the AI's brain) to the public for free. You download these files and run them on your own hardware. Zero API costs, zero internet required.

The most accurate way to do this in Python is a Two-Step Pipeline:

Step 1 (Extractive Filter): Use an extractive model (like Malaya.summarize.extractive.transformer) to reduce a long 1-hour transcript down to the top 20% most important sentences.

Step 2 (Abstractive Polish): Feed those "top sentences" into an abstractive model (like Malaysian-T5).

Why do this?

Speed: Abstractive models are slow on long text. Shortening it first makes the process 5x faster.

Accuracy: It prevents the abstractive AI from "wandering off" or hallucinating, because it only sees the most relevant facts


Stemming ✂️
Cuts words to a base form by removing endings, sometimes not a real word.
Example:
running → run
studies → studi (not a real word)
Lemmatization 📖
Converts words to their real dictionary base form (lemma) using grammar rules.
Example:
running → run
studies → study (real word)

In one line:
👉 Stemming = faster but rough
👉 Lemmatization = slower but more accurate
